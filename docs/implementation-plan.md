# План Реализации

Работать небольшими этапами. Не пытаться собрать весь маркетплейс сразу.

## Этап 0: Репозиторий И Окружение

- Установить PHP, Composer, PostgreSQL, Redis и Git.
- Инициализировать Laravel с Inertia, React, Tailwind и Vite.
- Добавить примеры окружения и инструкции локального запуска.
- Добавить базовый CI, когда Git будет доступен.

## Этап 1: Авторизация, Роли, Layout

- Реализовать регистрацию и вход.
- Добавить роли: customer, performer, moderator, admin.
- Добавить policies для защищенных зон.
- Собрать public, dashboard, order workspace и admin layouts.

## Этап 2: Публичные Страницы

- Главная.
- Список категорий.
- Каталог услуг.
- Страница услуги.
- Профиль исполнителя.
- Биржа заданий.
- Заглушки правил и политики конфиденциальности.

## Этап 3: Категории И Услуги

- CRUD категорий для админа.
- CRUD услуг для исполнителя.
- Модерация услуг.
- Опции услуг.
- Публичные фильтры и сортировка.

## Этап 4: Задания И Отклики

- CRUD заданий для заказчика.
- Модерация заданий.
- Публичная биржа заданий.
- Отклики исполнителей.
- Выбор отклика заказчиком.

## Этап 5: Заказы И Статусы

- Создание заказа из услуги.
- Создание заказа из выбранного отклика на задание.
- Заглушка платежей.
- Явные переходы статусов.
- Таймлайн заказа.

## Этап 6: Чат И Файлы

- Внутренний чат заказа.
- Загрузка файлов с ограничением типа и размера.
- Policies видимости файлов.
- Модерация текста и названий файлов.

## Этап 7: Антиувод Контактов

- Общий moderation checker.
- Блокирующие правила для чата, услуг, заданий, профилей, названий файлов и извлеченного текста.
- Админ-экран `moderation_flags`.
- Эскалация повторных попыток.

## Этап 8: Админка, Отзывы, Споры

- Админ-дэшборд.
- Управление пользователями.
- Просмотр заказов и споров.
- Создание отзывов после завершенных заказов.
- Базовое разрешение споров.

## Этап 9: Тесты И Полировка

- Тесты policies.
- Тесты переходов статусов заказа.
- Тесты правил модерации.
- Smoke-тесты публичных страниц.
- Проверка адаптива.
- Пустые состояния и ошибки.

## Отчет После Каждого Этапа

После каждого этапа фиксировать:

- что сделано
- какие файлы изменены
- как проверить
- следующий рекомендуемый шаг
