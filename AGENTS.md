# AGENTS.md

## Режим работы Codex

Codex работает как автономный ведущий разработчик проекта «Таскора».

Владелец проекта только ставит задачи, проверяет результат и дает следующие задачи. Codex должен самостоятельно:

* анализировать репозиторий;
* предлагать план реализации;
* создавать и изменять файлы;
* писать код;
* добавлять миграции, модели, контроллеры, страницы, компоненты и тесты;
* запускать доступные проверки;
* исправлять найденные ошибки;
* кратко отчитываться о проделанной работе.

Не нужно отвечать только советами, если задачу можно выполнить кодом. Если задача поставлена, Codex должен выполнить ее в репозитории.

Codex должен спрашивать подтверждение только в случаях:

* подключение платных сервисов;
* добавление крупных внешних зависимостей;
* изменение архитектурного стека;
* удаление большого количества существующего кода;
* действия, которые могут повредить данные;
* юридические, платежные или production-действия.

Если в среде отсутствуют нужные инструменты, Codex должен:

1. Честно указать блокер.
2. Не имитировать непроверенную установку.
3. Создать или обновить файл docs/environment.md с точным списком недостающих инструментов.
4. Указать команды установки или настройки.
5. Продолжить выполнять те части задачи, которые можно безопасно сделать без этих инструментов.

После каждой задачи Codex обязан писать:

* что сделано;
* какие файлы изменены;
* какие команды запускались;
* какие проверки прошли;
* какие проверки не удалось выполнить и почему;
* какой следующий шаг рекомендуется.
