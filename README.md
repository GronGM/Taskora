# Taskora

Taskora, интерфейсное название «Таскора», — русскоязычный маркетплейс задач и услуг. Сервис проектируется как модульный монолит: заказчик может купить готовую услугу или разместить индивидуальное задание, исполнитель — опубликовать услугу, откликнуться на задание и выполнить заказ внутри платформы.

Проект не копирует Kwork и Studwork по дизайну, текстам, структуре страниц или фирменному стилю.

## Текущее Состояние

Создан рабочий Laravel + Inertia + React каркас:

- Laravel-приложение установлено в текущий каталог;
- подключен Inertia middleware;
- подключены React, Inertia React, Vite и Tailwind CSS;
- создан общий публичный layout;
- создана главная страница на русском языке;
- реализованы вход, регистрация и выход;
- добавлены роли `customer`, `performer`, `moderator`, `admin`;
- добавлены role-protected dashboard-страницы;
- добавлены локальные seed-пользователи для проверки;
- добавлены категории, публичный каталог услуг, страницы категорий, страницы услуг и витрина исполнителей;
- добавлены локальные demo-услуги исполнителя;
- добавлено управление услугами в кабинете исполнителя: список, создание, редактирование, отправка на модерацию и архивирование;
- добавлена базовая защита от передачи контактов в текстах услуг и пакетах;
- добавлена модераторская очередь услуг: просмотр, публикация, отклонение с причиной и обработка флагов модерации;
- локальный первый запуск настроен на SQLite.

В проекте не подключены:

- реальный платежный шлюз;
- AI-функции;
- внешние платные API;
- Docker-зависимый запуск.

## Запуск

Установить PHP-зависимости:

```bash
composer install
```

Установить JS-зависимости:

```bash
npm install
```

Создать локальную SQLite-БД, если файла еще нет:

```bash
php -r "file_exists('database/database.sqlite') || touch('database/database.sqlite');"
```

Выполнить миграции и сидеры:

```bash
php artisan migrate:fresh --seed
```

Собрать frontend:

```bash
npm run build
```

Запустить локальный сервер:

```bash
php artisan serve --host=127.0.0.1 --port=8000
```

Открыть:

```text
http://127.0.0.1:8000
```

Для разработки frontend можно отдельно запустить:

```bash
npm run dev
```

## Локальные Тестовые Пользователи

Эти аккаунты создаются сидером только для локальной разработки. Не использовать такие email и пароль в production.

| Роль | Email | Пароль | Кабинет |
|---|---|---|---|
| Заказчик | `customer@taskora.local` | `password` | `/customer/dashboard` |
| Исполнитель | `performer@taskora.local` | `password` | `/performer/dashboard` |
| Модератор | `moderator@taskora.local` | `password` | `/moderator/dashboard` |
| Администратор | `admin@taskora.local` | `password` | `/admin/dashboard` |

Публичная регистрация разрешает выбрать только роли заказчика и исполнителя. Роли модератора и администратора назначаются только внутри платформы.

## Публичные Маршруты Каталога

| Маршрут | Назначение |
|---|---|
| `/` | главная страница с категориями и популярными услугами из базы |
| `/catalog` | общий каталог опубликованных услуг |
| `/catalog?category=word-i-dokumenty` | фильтр каталога по категории через query-параметр |
| `/catalog/{category:slug}` | публичная страница категории |
| `/services/{service:slug}` | публичная страница опубликованной услуги |
| `/performers` | публичная витрина исполнителей с опубликованными услугами |

## Маршруты Кабинета Исполнителя

Все маршруты требуют входа и роли `performer`.

| Метод | Маршрут | Назначение |
|---|---|---|
| `GET` | `/performer/services` | список услуг исполнителя |
| `GET` | `/performer/services/create` | форма создания услуги |
| `POST` | `/performer/services` | сохранение новой услуги |
| `GET` | `/performer/services/{service}/edit` | форма редактирования своей услуги |
| `PUT/PATCH` | `/performer/services/{service}` | обновление своей услуги |
| `POST` | `/performer/services/{service}/submit-review` | отправка услуги на модерацию |
| `POST` | `/performer/services/{service}/archive` | архивирование своей услуги |

Новая услуга создается в статусе `draft`. Исполнитель не может вручную выбрать публичный статус. Отправка на модерацию переводит услугу в `pending_review`. Изменение важных полей опубликованной услуги также возвращает ее на модерацию.

Статусы в интерфейсе:

| Технический статус | Русский статус |
|---|---|
| `draft` | Черновик |
| `pending_review` | На модерации |
| `published` | Опубликована |
| `rejected` | Отклонена |
| `archived` | В архиве |

## Защита От Передачи Контактов

При создании и редактировании услуги проверяются название, короткое описание, полное описание, названия и описания пакетов. Блокируются явные email, телефоны, Telegram, WhatsApp, VK, Discord, Skype, банковские карты и фразы вроде "напиши в тг", "давай напрямую", "оплата мимо сайта".

Если правило сработало, услуга не сохраняется, пользователь получает ошибку на русском языке, а срабатывание записывается в `moderation_flags`.

## Маршруты Модератора

Маршруты требуют входа и роли `moderator` или `admin`. Модераторская очередь не дает доступ к админским настройкам платформы.

| Метод | Маршрут | Назначение |
|---|---|---|
| `GET` | `/moderator/services` | список услуг со статусом `pending_review` |
| `GET` | `/moderator/services/{service}` | просмотр услуги на модерации |
| `POST` | `/moderator/services/{service}/approve` | публикация услуги |
| `POST` | `/moderator/services/{service}/reject` | отклонение услуги с причиной |
| `GET` | `/moderator/moderation-flags` | список открытых флагов модерации |
| `POST` | `/moderator/moderation-flags/{flag}/resolve` | отметка флага как обработанного |

Сценарий проверки:

1. Войти как `performer@taskora.local` / `password`.
2. Создать услугу и отправить ее на модерацию.
3. Выйти и войти как `moderator@taskora.local` / `password`.
4. Открыть `/moderator/services`, проверить услугу и опубликовать ее.
5. Убедиться, что опубликованная услуга появилась в `/catalog`.
6. Создать еще одну услугу, отклонить ее с причиной и проверить, что исполнитель видит причину в `/performer/services`.

Сидеры создают стартовые категории и demo-услуги только для локальной разработки. Эти данные нужны для проверки MVP-интерфейса и не должны использоваться как production-контент.

Основные категории:

- Тексты и документы
- Презентации
- Дизайн
- Сайты и разработка
- Маркетинг
- Бизнес-задачи

Demo-услуги создаются от `performer@taskora.local`:

| Услуга | URL |

## Переменные Окружения

Основные локальные значения:

```dotenv
APP_NAME="Таскора"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000

APP_LOCALE=ru
APP_FALLBACK_LOCALE=ru
APP_FAKER_LOCALE=ru_RU

DB_CONNECTION=sqlite

SESSION_DRIVER=database
CACHE_STORE=database
QUEUE_CONNECTION=database
FILESYSTEM_DISK=local

TASKORA_PLATFORM_FEE_PERCENT=15
TASKORA_CONTACT_GUARD_ENABLED=true
TASKORA_MODERATION_QUEUE_ENABLED=true
TASKORA_PAYMENTS_MODE=stub
```

Будущие настройки PostgreSQL оставлены в `.env.example` комментариями. На первом этапе PostgreSQL, Docker и Redis не требуются.
